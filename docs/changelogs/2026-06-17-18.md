## 3. AI Price Pipeline ‚Äî Full Replacement

Replaced the static seed + deterministic formula with a two-stage AI pipeline.

**Stage 1 ‚Äî Price Gathering (`priceGathering.ts`):**
- Checks MongoDB for similar observations (last 90 days)
- If fewer than 5 real observations found ‚Üí calls Tavily web search
- Tavily runs two queries (Hebrew + English), extracts ILS prices via regex
- Filters out irrelevant prices (5x ratio ‚Äî removes car/real-estate noise)
- Deduplicates and caps at 15 observations per search
- Saves results to MongoDB

**Stage 2 ‚Äî AI Analysis (`aiAnalysis.ts`):**
- Single OpenAI structured prompt receives listing + condition + all gathered prices + data sources
- AI decides the full verdict: `worth_it` / `maybe` / `avoid`, rating 1‚Äì5, confidence
- AI also writes the reasoning (summary, positives, concerns)
- Fallback to `maybe / low confidence` when OpenAI is unavailable

**Deleted:** `verdict.ts` (deterministic formula), `narrative.ts`, `static.ts` (static seed), `seed.ts`, `marketContext.ts`

## 5. Passive Market Data Collection

Built a system to grow the MongoDB database organically without user effort.

**Browse page collection:**
- Extension silently scans all visible listing cards as the user browses Facebook Marketplace
- Collects title + price for every card without any user action
- Batches results and sends to `POST /marketplace/observe` every 30 seconds
- Deduplicates by URL within each session

**Item page enrichment (Option A):**
- When a user clicks into any listing (not just when they Analyze), the extension silently saves the full listing ‚Äî title + price + **description** + URL
- Waits 2 seconds for Facebook's client-side render to finish, then captures
- `description` field added to `MarketObservation` shared type, MongoDB model, and all related code

**New backend endpoint:**
- `POST /marketplace/observe` ‚Äî accepts batches of up to 50 observations, saves to MongoDB
